# 🎨 Picasso, I Choose You!

Repaint your photo in the style of any artwork. Bring a photo and a piece of art —
a painting, a comic panel, a texture — and the app blends them into something new for
posters, covers, and prints.

Built as a learning project on the neural style transfer method of
**Gatys, Ecker & Bethge (2015)**, following **Prof. Mitesh Khapra's _Deep Art_
lecture** (CS7015, IIT Madras).

## What it does

A neural network trained to recognise objects turns out to be able to separate
*what* is in a picture (its **content**) from *how* it is painted (its **style**).
The app starts from a copy of your photo and repeatedly nudges it until its content
still matches your photo while its style matches the artwork you chose.

Every control is a plain-language dial over that process:

- **Style strength** — how strongly the artwork's look takes over your photo.
- **Detail** — the working image size; larger is sharper but slower.
- **Refinement passes** — how many times the picture is nudged toward the goal.
- **Texture scale** — borrow the artwork's fine grain or its bold shapes.

## How it works (short version)

A pretrained **VGG19** reads both images. Content is read from one deep layer's
activations; style is read from **Gram matrices** (how strongly features fire
together) across several layers. The generated image's *pixels* — not the network's
weights — are optimised to minimise:

```
L_total = α · L_content + β · L_style
```

The **Style strength** slider is the β/α ratio on a logarithmic scale.

## Run locally (uv)

Prerequisites: [uv](https://docs.astral.sh/uv/) (0.5.3+) installed.

```bash
git clone https://github.com/<you>/Picasso_I_Choose_You.git
cd Picasso_I_Choose_You
uv sync                          # creates .venv and installs everything (CPU Torch)
uv run streamlit run app.py      # launches the app
```

That's it — `uv sync` reads `pyproject.toml`, pulls the CPU-only PyTorch build, and
`uv run` executes inside the managed environment.

**Have a local GPU?** Point `torch`/`torchvision` at a CUDA index instead of
`pytorch-cpu` in `pyproject.toml` (e.g. `https://download.pytorch.org/whl/cu124`),
then `uv sync`. The app auto-detects CUDA and will use it.

## Deploy to Streamlit Community Cloud

Streamlit Cloud installs from **`requirements.txt`** (it runs uv under the hood, or
pip as a fallback). The committed `requirements.txt` already pins the **CPU-only**
Torch build, which is what keeps the install inside Cloud's memory limit.

1. Push this project to a **public** GitHub repo (the free tier requires public repos).
2. Go to [share.streamlit.io](https://share.streamlit.io), sign in with GitHub, and
   click **New app**.
3. Select your repo/branch and set the main file to `app.py`.
4. Under **Advanced settings**, set the Python version to **3.11**.
5. **Deploy.** The first build takes a few minutes to install PyTorch.

### Keeping requirements.txt in sync with uv

Whenever you change dependencies via `uv add`, regenerate the deployment file:

```bash
uv export --no-hashes --format requirements-txt -o requirements.txt
```

(You can then re-add the CPU index line if `uv export` omits it; the committed file
shows the expected format.)

## A note on speed and limits

Streamlit Community Cloud's free tier is **CPU-only with a modest RAM ceiling** (~1 GB
guaranteed) and can time out on long tasks. Neural style transfer is iterative
optimisation, which is genuinely slow on CPU, so the app ships with conservative
defaults (Detail 400 px, 150 passes). Keep **Detail ≤ 450** on the free tier, and
expect each run to take a few minutes.

**Faster alternative:** [Hugging Face Spaces](https://huggingface.co/spaces) supports
Streamlit natively and offers free, quota-based GPU (ZeroGPU), which can turn those
minutes into seconds. `app.py` runs there unchanged on a CPU Space; using the free GPU
takes one small extra step to wire up.
